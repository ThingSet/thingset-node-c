struct thingset_api
#include <thingset_internal.h>

Internal functions that have to be implemented separately for text and binary mode.

The correct API struct is assigned to the ThingSet context at the beginning of processing an incoming message.

All serialize functions return 0 or negative error code. If a negative error code is returned, an error message may have been stored already. If ts->rsp_pos == 0, the error message has to be generated at the end.

Also deserialize functions return 0 or negative error code, but never store any error response in the buffer.
